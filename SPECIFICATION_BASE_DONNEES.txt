1. TABLE: users (Utilisateurs)
================================================================================

- password_hash: VARCHAR(255) NOT NULL
- full_name: VARCHAR(100) NOT NULL
- phone: VARCHAR(20)
- role: ENUM('admin', 'teacher', 'parent') DEFAULT 'admin'
- avatar: VARCHAR(255)
- last_login: DATETIME
- is_active: BOOLEAN DEFAULT true
- created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP
- updated_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP

FONCTIONS:
- createUser(username, email, password, full_name, phone, role)
- authenticateUser(username, password)
- updateUser(id, data)
- deleteUser(id)
- getUserById(id)
- getUserByUsername(username)
- getAllUsers()

LIENS ET RELATIONS PRINCIPALES:
================================================================================

1. HIERARCHIE UTILISATEURS:
   users -> teachers (1:1)
   users -> students (1:1 via parents)

2. HIERARCHIE SCOLAIRE:
   academic_years -> classes (1:N)
   classes -> students (1:N)
   classes -> class_subjects (1:N)
   subjects -> class_subjects (1:N)

3. HIERARCHIE ENSEIGNEMENT:
   teachers -> subjects (1:N)
   teachers -> schedule (1:N)
   teachers -> exam_schedule (1:N)

4. HIERARCHIE EVALUATION:
   exams -> grades (1:N)
   students -> grades (1:N)
   students -> report_cards (1:N)
   report_cards -> report_card_details (1:N)

5. HIERARCHIE PRESENCE:
   students -> attendance (1:N)
   classes -> attendance (1:N via students)

6. HIERARCHIE FINANCIERE:
   students -> payments (1:N)

7. HIERARCHIE NOTIFICATIONS:
   users -> notifications (1:N)
   classes -> notifications (1:N)
   students -> notifications (1:N)

8. HIERARCHIE PERSONNALISATION:
   users -> user_preferences (1:1)

================================================================================

INDEX RECOMMANDES:
================================================================================

1. users: username, email, role
2. students: class_id, status, academic_year
3. attendance: student_id, date
4. grades: exam_id, student_id
5. payments: student_id, status, due_date
6. schedule: class_id, day_of_week
7. notifications: target_user_id, is_read, created_at
8. audit_logs: user_id, created_at, table_name

================================================================================

CONTRAINTES ET VALIDATIONS:
================================================================================

1. Notes: 0 <= score <= max_score
2. Présences: Une seule entrée par élève par jour
3. Grades: Une seule note par élève par examen
4. Schedule: Pas de conflit de professeur ou de salle
5. Payments: paid_amount <= amount
6. Academic Year: start_date < end_date
7. Students: enrollment_date >= date_of_birth + 5 years
